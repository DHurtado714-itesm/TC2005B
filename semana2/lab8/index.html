Preguntas a responder

Lab 1

¿Cuál es la diferencia entre Internet y la World Wide Web?

Internet es una red de redes que conecta a millones de computadoras alrededor del mundo. La World Wide Web es un servicio que se utiliza para acceder a la información que se encuentra en Internet.

¿Cuáles son las partes de un URL?

El URL está compuesto por tres partes: el protocolo, el nombre del servidor y el nombre del recurso. El protocolo es el que indica cómo se va a acceder al recurso. El nombre del servidor es el nombre de la máquina que contiene el recurso. El nombre del recurso es el nombre del archivo que contiene la información que se quiere acceder.

¿Cuál es el propósito de los métodos HTTP: GET, HEAD, POST, PUT, PATCH, DELETE?

GET: Solicita una representación de un recurso específico. Las peticiones que usan el método GET sólo deben recuperar datos. HEAD: Solicita una respuesta idéntica a la de una petición GET, pero sin el cuerpo de la respuesta. POST: Envia una entidad al recurso especificado, causando a menudo un cambio en el estado o efectos secundarios en el servidor. PUT: Reemplaza todas las representaciones actuales del recurso de destino con la carga útil de la petición. PATCH: Es utilizado para aplicar modificaciones parciales a un recurso. DELETE: Borra un recurso en específico.

¿Qué método HTTP se debe utilizar al enviar un formulario HTML, por ejemplo cuando ingresas tu usuario y contraseña en algún sitio? ¿Por qué?

El método POST se utiliza para enviar datos a un servidor para crear o actualizar un recurso. El método GET se utiliza para solicitar datos de un recurso específico.

¿Qué método HTTP se utiliza cuando a través de un navegador web se accede a una página a través de un URL?

El método GET se utiliza para solicitar datos de un recurso específico.

Un servidor web devuelve una respuesta HTTP con código 200. ¿Qué significa esto? ¿Ocurrió algún error?

El código 200 significa que la petición se ha completado con éxito. No ocurrió ningún error.

¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 404? ¿Por qué?

No es responsabilidad del desarrollador corregir el error 404. El error 404 significa que el servidor no pudo encontrar el recurso que se solicitó.

¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 500? ¿Por qué?

Sí es responsabilidad del desarrollador corregir el error 500. El error 500 significa que el servidor encontró una condición inesperada que le impide cumplir con la petición.

¿Qué significa que un atributo HTML5 esté depreciado o desaprobado (deprecated)? Menciona algunos elementos de HTML 4 que en HTML5 estén desaprobados.

Significa que el atributo ya no se debe utilizar. Los elementos de HTML 4 que en HTML5 están desaprobados son: font, center, big, strike, tt, frame, frameset, noframes, applet, basefont, dir, isindex, menu, s, u.

¿Cuáles son las diferencias principales entre HTML 4 y HTML5?

HTML5 es más semántico que HTML4. HTML5 es más fácil de usar que HTML4. HTML5 es más compatible con dispositivos móviles que HTML4. HTML5 es más compatible con CSS3 que HTML4. HTML5 es más compatible con JavaScript que HTML4.

¿Qué componentes de estructura y estilo tiene una tabla?

Las tablas tienen componentes de estructura y estilo. Los componentes de estructura son: table, tr, td, th, caption. Los componentes de estilo son: col, colgroup, thead, tbody, tfoot.

¿Cuáles son los principales controles de una forma HTML5?

Los principales controles de una forma HTML5 son: input, textarea, select, button, datalist, output, progress, meter.

Lab 3

Como ingeniero de software ¿cuál es tu recomendación sobre el uso de !important en un CSS?

No se debe utilizar !important en un CSS. Si se utiliza !important en un CSS, se está violando el principio de especificidad de CSS. El uso de !important en un CSS puede causar que el código CSS sea difícil de mantener.

Si se pone una imagen de fondo en una página HTML, ¿por qué debe escogerse con cuidado?

Se debe escoger con cuidado la imagen de fondo de una página HTML porque si la imagen es muy grande, la página tardará más en cargar. Si la imagen es muy pequeña, la imagen se verá pixelada.

Como ingeniero de software, ¿cuál es tu recomendación al elegir las unidades de un propiedad de estilo entre %, px y pt?

Se debe elegir las unidades de un propiedad de estilo entre %, px y pt dependiendo del tamaño de la pantalla. Si el tamaño de la pantalla es pequeño, se debe utilizar % o px. Si el tamaño de la pantalla es grande, se debe utilizar pt.

¿Por qué el uso de una versión minimizada del CSS mejora el rendimiento del sitio?

Elimina bytes innecesarios del CSS. Reduce el tiempo de carga de la página y mejora el rendimiento del sitio.
